# 가락몰 실내 길찾기 웹앱 (프로토타입)

서울 가락시장 가락몰 소매 구역의 **설치 불필요 모바일 웹 길찾기** 프로토타입입니다.
건물 곳곳의 QR코드를 스캔하면 해당 위치가 "현재 위치"로 열리고,
사고 싶은 품목을 **음성** 또는 **카테고리 버튼**으로 입력하면
가장 가까운 매장을 찾아 **SVG 평면도 위에 경로**로 안내합니다.

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 방문객용 화면 구조 (단일 페이지) |
| `styles.css` | 모바일 세로·큰 글씨·고령 친화 스타일 |
| `app.js` | 음성인식 · 매칭 · SVG 지도/경로 로직 |
| `stores.json` | **매장 데이터** (공공데이터 실매장 1,555개 + 데모용 정밀좌표 일부) |
| `locations.json` | **QR 위치 데이터** (판매동 4개 지점, 평면도 픽셀좌표) |
| `zones.json` | **구역 대표좌표** (판매동 B1·B2·3층은 공식 평면도 정합 완료) |
| `floorplans/` | **공식 층별 평면도 PNG 30장** + index.json(크기·통로·에스컬레이터 좌표) |
| `assets/` | 마스코트 이미지 (친환경무농이·신선이) |
| `admin-qr.html` | **운영자용 QR코드 생성·다운로드·인쇄 페이지** |
| `admin-map.html` | **운영자용 좌표 매핑 도구** (배치도 클릭 → 매장 x,y 입력) |
| `tools/convert_garak_csv.py` | **공공데이터 CSV → stores.json 변환기** |
| `qr-codes/` | **미리 만들어 둔 QR PNG 파일들** (`qr-<locId>.png`, 8개) |
| `vendor/qrcode.min.js` | QR 생성 라이브러리 (로컬 포함, 외부 의존 없음) |

## 실행 방법

### 가장 쉬운 방법 — 더블클릭 한 번 (macOS)
**`start.command`** 파일을 Finder에서 더블클릭하세요. 자동으로:
- PC의 와이파이 IP를 감지해 서버를 켜고
- QR 생성 페이지를 브라우저로 열며
- 폰으로 접속할 주소를 화면에 표시합니다 (종료: 터미널 창에서 `Ctrl + C`)

> 처음 실행 시 "확인되지 않은 개발자" 경고가 뜨면, 파일을 **우클릭 → 열기**로 한 번 허용하면 됩니다.

### 또는 명령어로 직접
> ⚠️ `index.html` 을 더블클릭(file://)으로 열면 브라우저가 JSON `fetch` 를 막아
> 데이터가 로드되지 않습니다. 아래처럼 **로컬 서버**로 여세요.

```bash
cd garakmall-wayfinding

# 파이썬3 (기본 권장)
python3 -m http.server 8421
#  → 브라우저에서  http://localhost:8421/

# 또는 Node 사용 시
npx serve .
```

### QR 위치 시뮬레이션
URL 파라미터 `?loc=` 로 현재 위치를 바꿉니다 (각 QR코드가 이 값을 가짐).
`locId` 는 `동-층-지점`(예: `g-b1-west`) 규칙으로 건물·층을 구분합니다:

```
http://localhost:8421/?loc=g-b1-west   # 판매동 지하1층 서측 입구 (기본값)
http://localhost:8421/?loc=g-b1-esc    # 판매동 지하1층 중앙 에스컬레이터 앞
http://localhost:8421/?loc=g-b1-east   # 판매동 지하1층 동측 입구
http://localhost:8421/?loc=g-3f-esc    # 판매동 3층 식당가 에스컬레이터 앞
```
`loc` 값이 없으면 첫 번째 위치(서측 입구)로 자동 설정됩니다.

### 발표 데모 시나리오 (권장 동선)
1. **같은 층 안내**: `?loc=g-b1-west` 진입 → 🍎 청과·과일 (또는 음성 "과일")
   → 판매동 지하1층 공식 평면도 위에 현재위치·통로 경로·매장 마커 표시
2. **층간 이동 안내**: 같은 위치에서 🦑 건어물·특산품 → "먼저 에스컬레이터로" 안내
   + 지하1층에서 에스컬레이터까지 경로 → 층탭 **3층** 전환 시 에스컬레이터→매장 경로 표시
3. 좌표 검수용 격자: URL에 `&debug=1` 을 붙이면 지도에 50px 격자·좌표 눈금 표시

> ⚠️ 판매동 1층(수산·건해·축산)은 공식 평면도가 없어 약식 안내도로 폴백됩니다.
> 발표 시연 동선에서는 제외하세요.
> 📱 음성인식·나침반은 발표 전 실기기(폰)로 1회 리허설 필수 (headless 검증 불가).

> 📱 음성인식(Web Speech API)은 **Chrome / Safari** 계열에서 동작하며,
> 마이크 권한 허용이 필요합니다. 미지원 환경에서는 카테고리 버튼만 활성화됩니다.

## 실물 QR코드 만들기 / 받기 (운영자)

방문객이 스캔할 **실제 QR코드**는 `admin-qr.html` 에서 생성하고, **이미지 파일로 받을 수 있습니다.**

### 미리 만들어 둔 QR (지금 바로 사용)
`qr-codes/` 폴더에 8개 위치의 QR이 **PNG 파일**로 들어 있습니다 (`qr-<locId>.png`).
바로 열어보거나 인쇄소에 넘길 수 있습니다.

> ⚠️ 이 PNG들은 데모용 임시 도메인 `https://garak.example.com` 으로 인코딩돼 있습니다.
> **실제 운영 도메인으로는 아래 절차로 다시 생성**해야 스캔 시 올바른 주소로 열립니다.

### 직접 생성·다운로드하는 법
1. 로컬 서버 실행 후 `http://localhost:8421/admin-qr.html` 접속
2. 상단 **"배포 기본 주소"** 칸에 이 앱이 올라갈 실제 도메인을 입력
   (예: `https://garak.example.com/index.html`) → QR이 즉시 갱신됨
3. `locations.json` 의 위치마다 `?loc=<locId>` URL을 담은 QR이
   **건물(관)·층별로 그룹지어** 자동 생성됨 (카드에는 QR 이미지 + 위치명만 표시)
4. **QR을 받는 방법 (택1)**
   - 카드의 **`📥 PNG 저장`** 버튼 → 해당 QR 한 장을 `qr-<locId>.png` 로 다운로드
   - 상단 **`📥 QR 전체 PNG 저장`** 버튼 → 모든 QR을 한꺼번에 다운로드
   - 상단 **`🖨️ 전체 인쇄 / PDF 저장`** → 인쇄하거나 PDF로 저장 (건물마다 새 페이지)
5. 받은 QR을 **해당 건물·층의 지점(정문·출입구·에스컬레이터 등)에 부착**

> 동작 원리: QR에는 `index.html?loc=gate2` 같은 URL이 인코딩됩니다.
> 방문객이 스캔하면 그 위치가 "현재 위치"로 설정된 길찾기 화면이 열립니다.
> 즉 **QR의 부착 위치 = `locations.json` 의 좌표**가 되도록 맞춰야 합니다.

## 실제 가락몰 데이터로 채우기 (공식 공공데이터 + 좌표 매핑)

> 현재 `stores.json`/`locations.json` 은 **더미(예시)** 입니다. 아래 절차로 실제 데이터를 넣으세요.

### 1단계 — 공공데이터포털에서 매장 정보 받기
**서울시농수산식품공사 "가락몰 시설정보"** (공식·무료, 약 1,142개 점포):
- 🔗 https://www.data.go.kr/data/15156388/fileData.do
- 제공 항목: 연번 · 상호명 · 활동장소(건물+층+호수) · 건물명 · 연락처 · 취급부류
- 다운로드에 로그인이 필요할 수 있습니다. CSV 파일을 받아 두세요.

### 2단계 — CSV를 stores.json 으로 변환
```bash
python3 tools/convert_garak_csv.py ~/Downloads/가락몰_시설정보.csv > stores.json
```
- 상호명→`name`, 건물명→`building`, 층→`floor`, 호수→`zone`, 취급부류→`categories` 로 변환됩니다.
- **`x`, `y` 는 `null` 로 비어 출력됩니다** (이 데이터에는 평면도 좌표가 없음).
- ⚠️ **재실행 주의**: 현재 `stores.json` 에는 데모 매장 8곳의 정밀좌표(x,y)가 직접 입력돼 있습니다.
  변환기를 다시 돌리면 이 좌표가 모두 사라지므로, 재변환 시 기존 파일을 백업해 두세요.
- 실제 CSV 헤더가 다르면 스크립트 상단의 `COLS` 매핑만 수정하세요.

### 3단계 — 구역 좌표 매핑 (네비의 마지막 조각)
매장 1,555개를 일일이 찍는 대신, **구역(건물+층) 48개**만 찍으면 매장이 자기 구역 좌표를 상속합니다.
```
http://localhost:8421/admin-map.html
```
1. 왼쪽 **구역 목록**에서 구역 선택 (좌표 없으면 주황 "미설정")
2. → 그 건물·층의 **공식 가락몰 평면도가 자동으로 배경에 깔립니다** (`floorplans/`)
3. 평면도에서 그 구역의 위치를 **클릭** → 대표 x,y 입력 (점으로 표시)
4. 48개를 다 찍은 뒤 **`💾 zones.json 내보내기`** → 받은 파일로 `zones.json` 교체

- `floorplans/` : 가락몰 공식 층별 평면도 30장(동코드 A~G) + `index.json`.
  동코드 매핑 — A:5관 · B:4관 · C/D:2관 · E:1관 · F:업무동 · **G:판매동**(지하 청과·채소).
- 더 정밀하게 하려면 매장(호수) 단위로도 찍을 수 있게 확장 가능(평면도에 호수까지 표시됨).

> ⚠️ 카카오맵 좌표(위경도/GPS)는 층 구분이 없고 실외 기준이라 실내 네비에 부적합합니다.
> 그래서 좌표는 위 도구로 **공식 층별 평면도 위에 직접 매핑**합니다.

## (참고) 더미 데이터 직접 편집하는 법

### 1) 매장 — `stores.json`
`stores` 배열에 매장을 추가/수정합니다. 좌표계는 평면도 SVG 기준 **0~1000**.

```json
{
  "id": "s09",
  "name": "○○청과",
  "building": "가락몰 1관",
  "zone": "A-15",
  "floor": 1,
  "categories": ["과일", "청과", "딸기"],
  "x": 640,
  "y": 300
}
```
- `categories` 에 넣은 단어가 곧 검색어와 매칭됩니다 (부분 일치 지원).
- `x`, `y` 는 실제 평면도에서의 상대 위치. 좌상단 (0,0) ~ 우하단 (1000,1000).

### 2) QR 위치 — `locations.json`
QR코드를 붙일 지점마다 항목을 추가합니다. **건물(`building`)·층(`floor`)별로 구분**해 저장하면
`admin-qr.html` 에서 건물·층 그룹으로 묶여 QR이 각각 생성됩니다.

```json
{ "locId": "b1-f1-fruit", "name": "청과동 입구", "building": "가락몰 1관", "floor": 1, "x": 200, "y": 120 }
```
- `building` 은 `stores.json` 의 `building` 과 같은 표기를 쓰세요.
- `locId` 는 `건물-층-지점`(예: `b1-f1-fruit`) 규칙을 권장합니다.
- → 이 QR코드가 가리킬 주소: `https.../index.html?loc=b1-f1-fruit`

### 3) 카테고리 버튼 — `app.js`
화면 상단 버튼은 `app.js` 의 `CATEGORIES` 배열에서 정의합니다.
`terms` 에 동의어를 넣으면 음성으로 다른 표현을 말해도 매칭됩니다.

```js
{ label: "정육·고기", terms: ["정육", "고기", "한우"] }
```

### 4) 평면도 모양 — `app.js`
`drawFloorPlanBackground()` 함수가 SVG 배경(외벽·통로·구역 블록)을 그립니다.
실제 가락몰 도면에 맞춰 사각형 좌표를 수정하면 됩니다.

## 동작 요약

1. QR 스캔 → `?loc=` 로 현재 위치 결정 (헤더에 표시)
2. 음성 또는 카테고리 버튼으로 품목 입력
3. 품목 취급 매장 필터 → 현재 위치에서 **직선거리 최단** 매장 선정
4. 결과 카드(매장명·관/구역·대표 품목·층 이동 안내) + 지도에 마커·경로선 표시
5. 다른 층 목적지면 층 탭으로 전환하여 확인

## 한계점 (프로토타입)

- **실시간 측위 미지원**: 현재 위치는 GPS/실내측위가 아니라 **QR 스캔 지점(고정)** 기준입니다.
  스캔 후 사용자가 이동하면 위치는 갱신되지 않습니다.
- **직선거리 근사**: 통로·벽·장애물을 고려하지 않은 단순 유클리드 거리로 최단 매장을 고릅니다.
  실제 도보 경로 길이와 다를 수 있습니다.
- **경로선 근사**: 층별 통로 중심선(corridor)에 투영한 경로로, 실제 매대 사이 세부 동선과는
  다를 수 있습니다. (통로 중심선은 판매동 B1·B2·3층만 정의됨, 그 외 층은 ㄱ자 폴백)
- **층 이동**: 에스컬레이터 지점을 경유하는 2단 안내(현재층 → 에스컬레이터, 목적층 → 매장)입니다.
- **좌표 범위**: 판매동 B1·B2·3층만 공식 평면도에 정합됐고, 판매동 1층(수산·건해·축산)은
  평면도가 없어 약식 안내도, 나머지 관(1~5관·업무동)은 임의 좌표입니다.
- **음성인식**: 브라우저·네트워크 의존(Web Speech API)이며 소음 환경에서 정확도가 떨어집니다.
  그래서 카테고리 버튼을 **항상** 함께 제공합니다.
- **QR 생성**: `admin-qr.html` 은 `vendor/qrcode.min.js`(로컬 포함, 외부 유료 API 아님)로
  QR을 그립니다. 인쇄·부착은 운영자가 수동으로 진행합니다.

## 향후 개선 방향(참고)

- 통로 노드 그래프 + A* 경로탐색으로 실제 도보 경로 안내
- BLE 비콘/Wi-Fi RTT 등 실내 측위 연동으로 이동 중 위치 갱신
- 매장 영업시간·재고·행사 정보 연동
- 실제 가락몰 도면 벡터화(층별 SVG)
